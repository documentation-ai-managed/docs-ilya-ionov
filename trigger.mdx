## Overview

The Subscriptions API lets you retrieve the current set of subscriptions in a single response. This page documents what is currently known about the `GET /subscriptions` endpoint and its response structure.

Authentication, pagination, and error behavior are not specified.

<Callout kind="info" collapsed="false">
  This page only documents fields and behavior that are explicitly known. Any unspecified behavior or additional fields are intentionally left out.
</Callout>

## Endpoint: GET /subscriptions

Use this endpoint to fetch the current subscriptions.

- **Method:** `GET`

- **Path:** `/subscriptions`

- **Query parameters:** not specified

- **Authentication:** not specified

- **Errors:** not specified

<Request show-lines="true" tabs={[]}>
  ```json
  curl https://api.example.com/subscriptions
  ```
</Request>

## Response format

The response body is a JSON object with a single known top-level field: `subscriptions`.

### `subscriptions` map

- **Type:** object / map

- **Key:** subscription id as a **string** (for example, `"1"`, `"2"`)

- **Value:** a subscription object with the fields described below

```json
{
  "subscriptions": {
    "1": {
      "subscriptionId": 1,
      "companySubscriptionId": "...",
      "status": "active",
      "periodStart": 1761988582,
      "periodEnd": 1761988582
    }
  }
}
```

<Callout kind="tip" collapsed="false">
  The same subscription id appears both as the map key (string) and as the numeric `subscriptionId` field inside each subscription object.
</Callout>

## Subscription object fields

Each entry in the `subscriptions` map has the following fields:

<ExpandableGroup>
  <Expandable title="subscriptionId" default-open="true">
    - **Type:** integer

    - **Description:** The numeric identifier of the subscription.

    - **Notes:** Mirrors the map key, but as a number instead of a string.
  </Expandable>

  <Expandable title="companySubscriptionId" default-open="false">
    - **Type:** string

    - **Description:** An external or company-level identifier for the subscription.

    - **Notes:** Exact format and semantics are not specified.
  </Expandable>

  <Expandable title="status" default-open="false">
    - **Type:** string

    - **Description:** The subscription status.

    - **Source:** Value from the protobuf `CompanySubscriptionStatus` enum.

    - **Example:** `"active"`

    - **Notes:** Only the literal `"active"` is confirmed; other possible enum values are not specified.
  </Expandable>

  <Expandable title="periodStart" default-open="false">
    - **Type:** integer

    - **Unit:** epoch seconds

    - **Description:** Start of the current subscription period.

    - **Notes:** Exact timezone and calendar semantics are not specified beyond being an epoch seconds timestamp.
  </Expandable>

  <Expandable title="periodEnd" default-open="false">
    - **Type:** integer

    - **Unit:** epoch seconds

    - **Description:** End of the current subscription period.

    - **Notes:** Represents `period_end + 1 day` at `00:00:00`. In other words, it is the epoch seconds timestamp for the midnight immediately after the billing period ends.
  </Expandable>
</ExpandableGroup>
